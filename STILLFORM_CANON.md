## 1 · WHAT STILLFORM IS

Stillform is a **metacognition practice**. Users build rich, specific mental representations of their own internal states, patterns, triggers, and reactions — the concept library the brain uses to perceive itself. Through repeated analytical practice, the library becomes more numerous, more granular, more accessible. Neuroplastic change accrues: analytical capacity becomes fast enough that pattern recognition feels like direct perception.

The result is **cognitive expansion**: sharper thinking, better strategic decisions, deeper self-understanding. Composure is one felt outcome that signals the practice is working — never the product framing.

Stillform is AI-assisted. The AI is the practice partner. It structures the analytical work, holds the boundaries of each session, and helps the user reach a takeaway that adds to the concept library. It does not regulate them, calm them, or fix them.

## 4 · EMOTIONAL HONESTY (v2 — added May 13, 2026)

**Metacognition is not the bypassing of emotion to reach action.** Felt experience — grief, vent, exhaustion, anger, presence with what is — IS the cognitive material being practiced on, not friction to clear. The practice makes room for the user to sit with what is, not just label and move it. Self-mastery is clear-eyed AND emotionally honest. Sentimentality and robotic stoicism are both drift.

**Audience promise, dual-axis:** the user gets sharper AND more whole. Composure as felt outcome. Emotional honesty as part of the practice. Neither at the cost of the other.

Science: Constructed emotion (Barrett 2017); affect labeling (Lieberman et al. 2007); dual-process grief (Stroebe & Schut 1999); polyvagal theory (Porges 2011) — empathic recognition creates the ventral vagal state required for higher-order cognition. Empathy is the on-ramp to metacognition, not separate from it.

Implementation in AI prompts: all three system prompts (CALM, CLARITY, HYPE) include a PRESENCE-FIRST mode for when the state arrives in grief, vent, exhaustion, or raw overwhelm. AI lands on what the user said with specificity, leaves space, ends. No pattern naming, no question, three sentences. Move past presence only when the user signals readiness.

## 5 · THE MECHANISM + SCIENCE SPINE

Mechanism: metacognitive practice → builds concept library → neuroplastic change → cognitive expansion. Regulation tools (breath, body scan) cut noise so the practice can land. They support the mechanism; they are not the mechanism.

**A proposal citing regulation literature only is missing the spine and fails Layer 0 audit.** Cite across the full spine:

- **Metacognitive training** — Flavell (1979); Nelson & Narens (1990); Schraw & Moshman (1995); Veenman et al. (2006); Frontiers in Psychology (2026) review; EEF metacognitive intervention data.
- **Adaptive vs maladaptive metacognition** — Wells (2009) metacognitive model of psychopathology; Hitchcock, Frank et al. (2024) meta-control account of rumination.
- **Emotional granularity** — Barrett (2001) coined; Kashdan, Barrett, McKnight (2015); Hoemann et al. (2021) experience sampling itself increases granularity; Suvak et al. (2011); Barrett (2017) constructed emotion theory.
- **Ecological Momentary Intervention** — Myin-Germeys et al. (2016, 2018). Stillform IS an EMI.
- **Neuroplasticity from practice** — Davidson (CHM); Lazar et al. (2005); Calderone et al. (2024); BDNF literature.
- **Self-mastery framework** — Bandura (1977) self-efficacy; Pearlin sense-of-mastery; performance psychology / cognitive fitness frameworks.
- **Deliberate practice** — Ericsson (1993); Ericsson & Pool (2016) *Peak*; Klein on expert intuition.
- **Cognitive bias** — Kahneman (2011); Tversky & Kahneman.
- **Affect labeling** (one supporting mechanism, not the foundation) — Lieberman (2007).
- **Cognitive reappraisal** (downstream, requires regulated baseline) — Ochsner & Gross (2005).
- **Regulation/somatic science** (supports the spine, is not the spine) — Porges polyvagal; Gerritsen & Band (2018); Khalsa et al. (2018); Lehrer et al. (2020).

## 6 · FAILURE MODES (the rumination guard)

Metacognition can produce rumination as easily as insight. Stillform must guard against rumination — which is NOT analysis itself, but a SPECIFIC FAILURE PATTERN within analysis.

**Rumination pattern:** unbounded, looping, circular, open-ended, judgment-coded.
**Productive analysis:** bounded scope, specific question, generative output, closure ritual, curious framing.

Design rules:

- **Bound every engagement** — specific question, defined scope, exit criteria, time limit
- **Invite curiosity, not judgment** — "what's the most accurate name for this?" not "why am I always like this?"
- **Produce takeaways, not commentary** — close with a specific insight, name, pattern, or move
- **Build closing rituals** — sessions END
- **Limit AI engagement length** — Reframe sessions cannot go indefinitely
- **Watch for meta-analysis affordances** — frequency-counting of states, surfacing pattern data, can trigger Type 2 rumination (Wells 2009)

Audit question for every surface: does this STRUCTURE analysis so it stays bounded, specific, generative, and closure-bound? Or does it invite the unbounded, looping, open-ended pattern that becomes rumination?

If a feature is bounded and generative — it serves the practice regardless of how analytical it is. If unbounded or open-ended — it trains rumination regardless of how "thoughtful" it appears.

## 7 · ARCHITECTURE — THE SPINE

**The universal session spine: Notice → Reframe → Close.** Every practice surface routes through this architecture.

- **Notice** — name what's present (feel-state chip, granularity prompt, optional precision label)
- **Reframe** — AI-assisted metacognition; modes auto-routed from feel state + input content (no mode buttons): excited/focused → hype; spiral language → clarity; everything else → calm
- **Close** — takeaway, optional save, library updated, session ends with closing language

**Three engines run in parallel:**

**Engine 1 — Retention (in-flow):** Mirror Strip (daily refresh), micro-achievements at every close, visible Roadmap from day 1.

**Engine 2 — Application Layer (life-bridge):** Today's Brief, Pre-event Brief, Move card, Scripts, EOD artifact, and User-Invented Move Library (queued build). These are takeaways FROM session work, not parallel entry points to practice. Move and Scripts live behind the Support Sheet, not on the home screen.

**Engine 3 — Trigger Profile (external risk mitigation):** capture/recognize recurring situations that produce the same patterns. Adaptive metacognition; never frequency-counting.

PracticeSurface cards route entry through the spine via `stillform_practice_entry_context`. Seven source values: default retrieval, spaced, pre-event, pre-mortem, open-recall, pre-sleep, deep-revisit.

Path A (decided May 7, 2026) consolidated Engine 2 takeaways into single-surface artifacts rather than parallel cards. Detail in `STILLFORM_ENGAGEMENT_ARCHITECTURE.md`.

## 8 · AUDIT PHILOSOPHY — PRIME DIRECTIVE

**EVERYTHING NEEDS INTEGRITY. NO FLUFF. NO FABRICATION. NO PATCHES. NO ASSUMPTIONS. NO DRIFT. NO FRAMING DRIFT.**

This is the lens. The nine layers below are the mechanism. Passing audit is not the goal — a correct, grounded, framing-aligned, verified system is the goal. When they diverge, the audit is wrong.

**Genesis:** the audit philosophy was created May 7, 2026 after Practice Signals shipped broken with 35/35 pre-commit audits passing — clean code, broken UI. Code hygiene is necessary but insufficient.

**How Arlin uses this:** she doesn't need to read code. She asks Claude the audit questions. Every question requires evidence (screenshot, grep, flow walkthrough) — not "trust me." If Claude cannot show the evidence, the audit didn't happen.

**The nine layers (run in order):**

0. **Framing audit** — verify against this doc before any other check
0.5. **Document context** — read the canonical docs that already answer the design questions
0.6. **Flow ground truth** — read the canonical app flow before designing user-facing surfaces
1. **Pre-existence** — before writing code (existing surfaces, science, worst-case paths)
2. **Code hygiene** — while writing
3. **Behavior** — after writing, before staging
4. **Voice + brand** — after writing, before staging
5. **Visual coherence** — phone screenshot, the irreducible final gate
6. **Self-skepticism** — continuous
7. **Recovery** — what happens when something slips through

**Audit discipline scales to risk.** Surgical contained changes verified by automated checks (syntax, preflight green) flow forward. Multi-file refactors touching live data or architecture hold for verification. Universal gating creates the bounce pattern; risk-scaled gating doesn't.

Detailed question lists for each layer live in `STILLFORM_AUDIT_PHILOSOPHY.md`.

## 10 · OPERATING RULES (locked decisions)

- **Nothing is post-launch.** Every promised feature ships at launch — launch is max attention; deferred features don't get the lens. NEVER use "post-launch," "future commit," "later," "deferred," "next phase" framing in code, commits, or scope. If infrastructure is needed, build it OR ship via alternative path. Enhancements to launch-complete features = fine. Deferring promised features = not.
- **Composure is a marketable felt outcome, NEVER the product headline.** Headline framing = self-mastery / metacognition. Composure shows up in testimonials, screenshots, what users say. Not in the positioning sentence.
- **Spine integration is the architecture.** Every practice surface through Notice → Reframe → Close. No parallel entry points to practice. Move and Scripts live behind Support Sheet.
- **Notifications tied to real cues only.** Calendar hits, EOD window, named precommitment. No "you haven't journaled in 3 days" prompts (rumination affordance).
- **Layer 0 before design.** Read framing law first. If the proposal doesn't fit framing, redesign before writing code.
- **No copy rewrites until architecture lands.** Don't paint walls before framing is up.
- **Verify before audit-claiming.** Never claim a problem exists in code, prompts, or copy without first viewing the actual source. No audit claims on unread files.
- **Symmetry across processing types.** Direction given for "the app" applies to whole app, not one type. Claude proactively audits every change for whether the equivalent should apply to the other type. Default question: does this need to work for both types?
- **Bobby is paper-only.** Robert Matthew Geismar is a name on the LLC paperwork only. NOT involved in development. All code changes are Claude or Cursor, prompted by Arlin only. Never attribute code to Bobby.
- **Pull `Stillform_Master_Todo.md` fresh from the repo before claiming launch state or remaining scope.** Compaction summaries are stale by design. Repo docs are truth.
- **Comms on overrides:** if Claude needs to override a setting/decision/rule, Claude ASKS first. Arlin does the override; Claude flags.
- **Single consolidated sign-off per change.** Don't fragment into micro-asks. One plan, one yes/no, one execution.
- **Integrations protect the practice. They are not the practice.** Calendar, health data, voice, focus modes, smart event detection, screenshot import, watch biometrics — these clear noise so the metacognition work can land. They are protective infrastructure. Stillform is never positioned as a personal assistant, a productivity tool, or a wellness service. Every integration must answer: does this remove friction that prevents the practice, or does it become the product? If the latter, redesign.
- **Reduce cognitive load on everything that is not the practice.** Fewer meta-decisions = more capacity for the work itself. Even 30 seconds of friction removed is 30 seconds the user can spend on themselves (Arlin's framing, May 14, 2026). Filter every feature through: does this subtract decisions the user otherwise has to make? Anything that ADDS meta-decisions to non-practice surfaces fails the filter. The voice is red-carpet/prestige/concierge — the system handles logistics so the user does the practice; never buddy-buddy or wellness-coach.
